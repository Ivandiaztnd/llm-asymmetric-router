01 — Architecture

The Dual-Node Nexus

Two dedicated Ollama instances, each tuned for a distinct workload class. A Python orchestrator intercepts every prompt and routes it before a single token is generated.

NODE // ALPHA

⚡ GPU — Interactive

Low-latency inference for real-time operator interaction, strategy, and Q&A.

  • backendOllama + Vulkan
  • port11434
  • context2048 tokens
  • modelGemma 2:2b Q4_K_M
  • optimised forTTFT < 300ms
NODE // BETA

🧠 CPU — Batch

High-context processing for logs, scan results, CVE correlation, SmartChunking.

  • backendOllama CPU (llama.cpp)
  • port11435
  • context4096+ tokens
  • modelGemma 2:2b Q4_K_M
  • optimised forThroughput / large ctx

02 — Router Design

Two-Stage Decision Pipeline

The router operates synchronously before inference. Every prompt passes through two stages before a node endpoint is selected.

Stage 1

Load Monitor
Poll VRAM via rocm-smi.
VRAM > 90% or TTFT > 800ms → force CPU route

→

Stage 2

Semantic Classifier
Pattern-match against configurable JSON ruleset.
Weighted scoring: multiple matches per class.

→

Dispatch

Node Endpoint
Async HTTP call via httpx.
Auto-fallback if node unreachable.

Trigger Keywords Route Rationale
DEBUG · LOG · SCAN_RESULT · NMAP · NUCLEI · CVE-* CPU — Beta High token count, latency-tolerant
EXPLOIT · STRATEGY · QUESTION · EXPLAIN · ANALYZE GPU — Alpha Low token count, latency-critical
CHUNK · SUMMARIZE · COMPRESS · BATCH CPU — Beta Batch semantics, large context
No match + prompt > 800 tokens CPU — Beta Length-based fallback
No match + prompt ≤ 200 tokens GPU — Alpha Short prompt default
Python — router usage 
from router import AsymmetricRouter, NodeConfig, RouterConfig
import asyncio

# Configure nodes
alpha = NodeConfig(endpoint="http://localhost:11434", model="gemma2:2b", context_window=2048)
beta  = NodeConfig(endpoint="http://localhost:11435", model="gemma2:2b", context_window=4096, backend="cpu")
cfg   = RouterConfig(vram_threshold=0.90, ttft_threshold_ms=800)

router = AsymmetricRouter(alpha, beta, cfg)

async def main():
    # Auto-routed to GPU — short interactive prompt
    r1 = await router.dispatch("Explain CVE-2024-1234 exploitation vector")

    # Auto-routed to CPU — large log batch
    r2 = await router.dispatch("SCAN_RESULT: [10K token Nmap output]")

    print(router.stats())
    # {'total_requests': 2, 'alpha_requests': 1, 'beta_requests': 1, 'avg_ttft_ms': 287.4}

asyncio.run(main())

04 — Quick Start

Up and Running in 5 Minutes

01

Clone & Install

Clone the repo and install Python dependencies via pip.

02

Pull Model

Pull Gemma 2:2b (or any compatible 2–7B model) via Ollama.

03

Launch Nodes

Start two Ollama instances on ports 11434 (GPU) and 11435 (CPU).

04

Configure & Route

Edit config.yaml, import AsymmetricRouter, call dispatch().

bash — full setup 
# 1. Clone
git clone https://github.com/ivandiaztnd/llm-asymmetric-router
cd llm-asymmetric-router
pip install -r requirements.txt

# 2. Pull model
ollama pull gemma2:2b

# 3. Node Alpha — GPU/Vulkan on port 11434
OLLAMA_HOST=0.0.0.0:11434 ollama serve &

# 4. Node Beta — CPU only on port 11435
OLLAMA_HOST=0.0.0.0:11435 CUDA_VISIBLE_DEVICES="" ollama serve &

# 5. Optional: Qdrant for persistent RAG
docker run -d -p 6333:6333 qdrant/qdrant

# 6. Run demo
python -m router --demo
